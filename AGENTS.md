# Контекст проекта

## Стек
- Бэкенд: .NET 10, C#, ASP.NET Core Web API
- Фронтенд: Next.js 14, TypeScript, Tailwind CSS
- БД: PostgreSQL 16
- Кэш: Redis (только для сессий)
- Файлы: локальная файловая система (до внедрения MinIO)
- Деплой: один Docker-контейнер, docker-compose

## Архитектурные правила
- Монолит, Clean Architecture
- Общая БД, схема public (временная)
- REST API, JSON
- Аутентификация: JWT без refresh (MVP)
- Никаких message brokers, gRPC, WebSocket в MVP
- Все данные на русском, но код и комментарии на английском
- Код должен быть простым, читаемым, без преждевременной оптимизации

## Принципы
- Простота превыше всего
- Не строить инфраструктуру, которой нет в требованиях
- Всегда генерировать миграции EF Core
- Обрабатывать ошибки единообразно (Result<T> паттерн)

## Запрещено
- Использовать микросервисные паттерны
- Добавлять технологии не из стека
- Генерировать код с несуществующими NuGet-пакетами
- Создавать избыточные абстракции

## Уточнение
если требование неясно — спроси, не выдумывай
