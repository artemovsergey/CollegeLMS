Проведи аудит плана на разработку LMS системы для колледжа. Разработка будет вестись одним человеком с помощью AI агентов (Claude или ChatGPT).

Также на основе анализа задания надо понять, как лучше организовать работу с агентами, какие дополнительные настройки сделать и предоставить данные. 

Стек разработки:  Postgres,  .NET,  Next.js,  Nginx
Архитектура: монолитное приложение

При разработке будет использован docker-compose. Развертывание будет на отдельном VPS через СI/CD Github Action

План действий:

- cначала надо разработать модели  для  API,  разработка будет  вестись через подход CodeFirst  путем создания миграций  с помощью  EF.  После выполнения миграций  будет  создана база данных  на сервере.
- после определения  моделей  сосредоточимся на создании  API
- после  того, как  API будет готово  надо будет создат клиенты на  Next .js и  ReactNative

Основные сервисы системы (это не микросервисы, это общее понятие):

Базовые сервисы (5 сервисов):

 - сервис сайта колледжа
 - сервис аутентификации и авторизации
 - сервис расписания занятий
 - сервис тестирования
 - сервис учебного процесса

Опциональные сервисы:

 - сервис электронного журнала 
 - сервис уведомлений
 - сервис генерации отчетов
 - сервис трасляции
 - сервис безопасности
 - сервис логгирования
 - сервис резервного копирования

1. Сервис сайта колледжа (SiteService)

Базовые требования:
- перенос данных с текущего сайта example.com
- сайт будет написан на Next.js. Колледж часто выкладывает новости, события.
- использовать компоненты Tailwind (версия 4)
- дизайн должен соответствовать цветам логотипа колледжа. Логотип и основные цвета прилагаются
- адаптивность сайта

текущая структура сайта и данные для анализа (домен для примера example.com):
 
http://example.com/sitemap.xml
http://example.com/sitemap-misc.xml 
http://example.com/category-sitemap.xml
http://example.com/page-sitemap.xml

все типы контента и маршруты 
http://example.com/wp-json/wp/v2/

http://example.com/wp-json/wp/v2/categories?per_page=100
http://example.com/wp-json/wp/v2/posts?per_page=100
http://example.com/wp-json/wp/v2/pages?per_page=100
http://example.com/wp-json/wp/v2/tags?per_page=100
http://example.com/wp-json/wp/v2/users?per_page=100

для  определения меню:
http://example.com/wp-json/wp/v2/pages?per_page=100&_fields=id,parent,title,slug,link


2. Сервис аутентификации и авторизации (AuthService)

Базовые требования:
- авторизация по логин и паролю
- ролевая модель: админ, преподаватель, студент, диспетчер.


3. Сервис расписания занятий (ScheduleService)

Расписание формирует диспетчер вручную, данные получаются в виде файлов для каждой группы и преподавателя. Пример файлов прилагается

Базовые требования:
- просмотр расписания по группе, преподавателю, аудитории.  
- редактирование (только диспетчер, админ).
- можно смотреть расписание на день, неделю, месяц, год, календарный вид
- экспорт расписания в pdf, excel

4. Сервис учебного процесса (LearningService)

Референсы: udemy, cousera, moodle, stepik

Базовые требования:

- управление студентами
- управление группами
- управление преподавателями
- создание учебных курсов (лекции, практики, тесты).  
- личные кабинеты студентов и преподавателей
- загрузка файлов (PDF, PPT, doc,docx, mp4)

18. Сервис интеграции с мессенджерами (AgentBridge)

Базовые требования:
- бот VK Max (Макс): просмотр расписания по группе/преподавателю, inline-кнопки навигации
- бот Telegram: интерфейс для взаимодействия с AI-агентом (OpenCode)
- интеграция с OpenCode Server API (HTTP + SSE)
- диспетчер может управлять расписанием через веб-приложение и рассылать уведомления группам через ботов
- запросы разрешений агента прокидываются в мессенджер (inline keyboard Allow/Deny)
- конфигурация через env vars, Docker контейнер в docker-compose

Стек: .NET 10, Telegram.Bot NuGet, VK Bots API (REST)
